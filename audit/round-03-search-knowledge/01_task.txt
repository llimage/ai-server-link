三、实现要求：Knowledge Ingestion
==================================================

请把 ingestion pipeline 与数据库状态联通：

流程：
1. upload
2. parse
3. chunk
4. index
5. publish

要求：

1. knowledge_sources
- upload 时创建 source
- 状态至少包含：
  - uploaded
  - parsed
  - chunked
  - indexed
  - published
  - failed

2. knowledge_documents
- parse 后创建 document
- title / metadata / status 落库

3. knowledge_chunks
- chunk 后写入数据库
- chunkIndex / chunkText / tokenCount / metadata 落库

4. publish
- publish 后可被 search.query 检索
- 未 publish 的内容不应进入正式检索结果

5. 错误处理
- 任一步失败时 source/document 状态应更新为 failed
- 保留错误信息到 metadata 或 log
